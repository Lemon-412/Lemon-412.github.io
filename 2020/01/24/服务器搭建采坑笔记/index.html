服务器搭建采坑笔记

Posted on 2020-01-24

基于V某技术y的服务器搭建采坑笔记

date: 2020-01-24
背景：目前S某技术S以及S某技术R负责人已经被请去喝汤停止维护相关项目。
导火索：去年国庆期间自己因为太浪被多次ban掉IP和端口，一直临时借用室友的同技术产品，但流速低到吓人，打开某G搜索网站需要2秒以上的时间。
又由于对某G网站的需要和对大千世界的好奇，并为今后可能到来的某G公司面试做好准备，拥有更高安全系数的V某技术y就进入视线了。

在编写本博客时，我牢记以下几点：

  • 热爱学习，热爱祖国，怀揣好奇的心，探索未知的世界
  • 服务器只供个人学习生活使用，不超出个人日常使用，不谋取任何利益
  • 不交流、不传播相应技术；不出租、出售相应产品，不谋取任何利益

1. SSH报错REMOTE HOST IDENTIFICATION HAS CHANGED


这是由于重装服务器更换Unbuntu18.04后再次SSH同一个IP地址造成的安全警告，只需删除相应的host记录即可（即图中host key文件目录下的known_hosts文件）
另外，重装系统会使某ID和PORT发生改变，配置json时踩了坑debug了好久。

2. 远程报错Failed to fetch release information. Please check your network and try again.


玄学问题，ssh连得上，ping某G网站也通，apt-get update也完全ok，但运行bash就是不行（即使本地拉过去也不行）。
后续更换了支持ipv4的服务器就没有出现过相关问题。v某r是按小时收费的，很贴心。只花了一美分就退掉了服务器。

3. 本地Chrome浏览器无法正常访问，Edge却可以

这似乎是由于Chrome走的路线和其他软件不太一样，不走系统某理，需要额外安装插件，如S某a。
如果安装相应crx出现问题，可以修改后缀为.zip解包后载入。
走S某技术5到127.0.0.1:1080或走H某技术L到127.0.0.1:8080，并启用相应模式即可。

4. 频繁出现网站意外终止连接，和503报错

网络时好时坏，好的时候一切正常，但有大约三分之一的概率打不开网站，Chrome报网站意外终止连接，或503 Service Unavailable，很短一段时间后又恢复正常。
后续检查发现是alterid近远端不匹配产生的问题，将alterid设置为正确的64重启电脑即可。

除去以上问题，大多数问题都能搜索到相应的解决办法。
整体来说，搭建服务器比搭hexo博客要省心一点，很多流程都已经自动化实现了。
